3) (Optional) Enhance public/app.js with state classes (no logic changes)

This only adds/toggles CSS classes on the orb and keeps your existing functions.
Find where you already set status/live, and add these helpers near the top:

// Add below your existing consts (dotEl already exists in your app)
function setOrbState(state){
  // states: 'idle' | 'connecting' | 'listening' | 'speaking' | 'error'
  const states = ['idle','connecting','listening','speaking','error'];
  states.forEach(s => dotEl.classList.remove(s));
  if (state) dotEl.classList.add(state);
}

Then sprinkle these in your existing flow (examples):
	•	After user clicks Start and before SDP is posted:

setOrbState('connecting');

	•	Once the data channel opens and you send session.update (ready to talk):

setOrbState('listening');

	•	If you detect the agent is speaking (optional hook if you parse events):

setOrbState('speaking');

	•	On normal stop/disconnect:

setOrbState('idle');

	•	On error/ICE failure:

setOrbState('error');

If you don’t have event hooks, it’s still fine—connecting → listening on connect already makes it feel alive.
